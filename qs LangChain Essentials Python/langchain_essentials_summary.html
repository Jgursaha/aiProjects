Core Architecture: The ReAct Agent

The centerpiece of this course is the ReAct (Reasoning and Acting) Agent, built on top of LangGraph.

  • The Loop: The model reasons about a request, uses tools to take action, gets observations, and loops until it can provide a final answer.
  • The Primitive: You can build this easily using create_agent from langchain.agents.

📓 Notebook Breakdown

1. The Fast Agent (L1_FastAgent)

  • Goal: Build an agent capable of exploring a SQLite database (Chinook.db) without prior schema knowledge.
  • Key Concept: The agent runs in a loop. If a SQL tool returns an error, the agent reads the error, self-corrects, queries the table names, and tries again.
  • UI: Taught how to visually trace and debug the agent's multi-step tool calls using the LangGraph Studio UI.

2. Models and Messages (L2_Models_And_Messages)

  • Messages as Circulation: Information flows through the agent via standard message types:
    • SystemMessage: Defines the core instructions and persona.
    • HumanMessage: The user's input/query.
    • AIMessage: Responses from the model, including tool call requests.
    • ToolMessage: The data returned from a tool execution.
  • Scratchpad: Messages aren't passed node-to-node; they append to a persistent scratchpad shared across the graph.

3. Streaming (L3_Streaming)

  • Problem: Agentic loops can take a long time to return a final answer.
  • Solutions (Stream Modes):
    • values: Emits state updates after every step.
    • messages: Streams the LLM output token-by-token.
    • custom: Allows emitting custom progress streams from inside tools using StreamWriter.

4. Tools (L4_Tools)

  • Action Generators: Standard Python functions wrapped in a @tool decorator.
  • Discovery: The LLM decides when and how to use a tool based entirely on the function's name, type hints, and docstring.
  • Optimization: Setting parse_docstring=True explicitly parses Google-style docstrings into strict metadata schemas for the LLM.

5. Tools with MCP (L5_Tools_w_MCP)

  • Model Context Protocol: An open standard that allows agents to use tools hosted outside of the Python process.
  • Usage: Utilized MultiServerMCPClient to connect to a local npx server, loading external tools directly into the LangChain agent.

6. Memory (L6_Memory)

  • Context: RuntimeContext is used to inject static dependencies (like DB connections) into tools.
  • Short-Term Memory: Fix amnesia by adding MemorySaver() (checkpointing) and passing a thread_id in the config to bind conversation state.

7. Structured Output (L7_StructuredOutput)

  • Goal: Forcing the LLM to reply with strict, predictable JSON data for computer systems to read.
  • Usage: By providing a response_format (like a Pydantic Model or a TypedDict), the agent outputs validated, structured data instead of raw text.

8. Dynamic Prompting & Middleware (L8_Dynamic_Prompt)

  • Middleware: Allows inserting custom code to intercept agent steps.
  • Dynamic Prompts: Use middleware functions to conditionally build and swap System Prompts on the fly based on runtime context (e.g. is_employee flag).

9. Human in the Loop (HITL) (L9_HITL)

  • Safety & Control: Adding interrupt breakpoints to the agent.
  • Usage: Pausing the graph to require manual human approval, rejection, or edits before executing sensitive tool calls.
